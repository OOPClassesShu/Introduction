# Занятие 3

## Статические поля и методы класса

В объектно-ориентированном программировании на C++ ключевое слово static применимо к членам класса и придаёт им особые свойства. \
Статические члены принадлежат не отдельным объектам, а самому классу. Они существуют в единственном экземпляре и доступны даже без создания объектов. 


**Статические поля (переменные класса)**
```
class MyClass {
public:
    static int count;   // объявление статического поля
};
```
Определение и инициализация: Статические поля должны быть определены вне класса (обычно в .cpp файле), \
чтобы под них была выделена память. Исключение — статические поля которые можно инициализировать прямо в классе, \ 
если они константные.

```
// MyClass.cpp
int MyClass::count = 0;   // определение и инициализация
```

inline static
```
class MyClass {
public:
    inline static int count = 0;   // C++17: определение и инициализация прямо здесь
};
```

**Статические методы**

Статические методы также принадлежат классу, а не объекту. Они не имеют указателя this и могут обращаться только к статическим членам \
(полям и другим статическим методам). Объявляются с ключевым словом static.
```
class MyClass {
public:
    static int getCount() { return count; }   // статический метод
    static void increment() { ++count; }
};
```
Определение статического метода вне класса не требует повторения static:
```
int MyClass::getCount() { return count; }   // правильно
```

**Доступ к статическим членам**

Обращаться к статическим членам можно:
- через имя класса с оператором разрешения области видимости ::;
- через любой объект класса (хотя это не рекомендуется, так как создаёт иллюзию принадлежности объекту).
```
MyClass::count = 5;                // обращение через класс
MyClass obj;
obj.count = 10;                    // тоже допустимо, но сбивает с толку
int x = MyClass::getCount();       // вызов статического метода через класс
int y = obj.getCount();            // вызов через объект (работает, но нежелательно)
```

**Особенности статических методов**

- Не имеют доступа к нестатическим полям и методам (нет this).
- Не могут быть const, volatile или virtual (виртуальными могут быть, но это редкость и обычно не имеет смысла, так как вызов не зависит от объекта).
- Могут быть перегружены.
- Могут быть объявлены как public, protected или private – как обычные методы.

## Примеры использования

Счетчик созданных объектов
```
#include <iostream>

class Widget {
private:
    inline static int totalCount = 0;   // C++17
public:
    Widget() { ++totalCount; }
    Widget(const Widget&) { ++totalCount; }
    Widget(Widget&&) noexcept { ++totalCount; }
    ~Widget() { --totalCount; }

    static int getCount() { return totalCount; }
};

int main() {
    std::cout << Widget::getCount() << '\n'; // 0
    Widget w1;
    Widget w2 = w1;
    std::cout << Widget::getCount() << '\n'; // 2
    {
        Widget w3;
        std::cout << Widget::getCount() << '\n'; // 3
    }
    std::cout << Widget::getCount() << '\n'; // 2
}
```

Доступ к приватным статическим полям через статические методы
```
class Application {
    static bool isInitialized;
public:
    static void init() { /* ... */ isInitialized = true; }
    static bool initialized() { return isInitialized; }
};

bool Application::isInitialized = false;
```
Еще:
- Константы класса
- Фабричный метод
- Реализация паттерна Singleton (одиночка)

  
# Указатель this

**this** — это ключевое слово в C++, которое представляет собой указатель на текущий объект, для которого вызван нестатический метод. \
Внутри любого нестатического метода класса неявно доступен указатель this, который содержит адрес объекта, с которым мы работаем. 

Тип указателя this зависит от контекста:

- В неконстантном методе: Class* const this — константный указатель на неконстантный объект (сам указатель изменить нельзя, но можно изменять объект).
- В константном методе: const Class* const this — константный указатель на константный объект (нельзя изменять ни указатель, ни объект через него).

Основные цели использования this:

- Разрешение конфликтов имён между параметрами метода и полями класса.
- Возврат текущего объекта из метода (для цепочек вызовов — fluent interface).
- Сравнение с другим объектом (например, проверка на самоприсваивание).
- Доступ к полям и методам объекта внутри метода (хотя обычно это неявно).
- Передача указателя на текущий объект в другие функции или методы.

Как работает this

Каждый нестатический метод класса при компиляции получает скрытый параметр — указатель на объект, для которого он вызван. 
Когда мы пишем obj.method();, компилятор преобразует это в нечто вроде method(&obj);. 
Внутри метода все обращения к полям и методам по умолчанию разрешаются через этот указатель, т.е. field становится this->field.

Пример неявного использования:
```
class Point {
    int x, y;
public:
    void setX(int value) {
        x = value;            // на самом деле this->x = value;
    }
};
```

Примеры использования

Разрешение конфликтов имён

```
class Person {
    std::string name;
public:
    void setName(const std::string& name) {
        this->name = name;   // поле name = параметр name
    }
};
```

Возврат текущего объекта из метода

```
class Accumulator {
    int value;
public:
    Accumulator& add(int x) {
        value += x;
        return *this;        // возвращаем ссылку на текущий объект
    }
    Accumulator& multiply(int x) {
        value *= x;
        return *this;
    }
    int get() const { return value; }
};

int main() {
    Accumulator acc;
    int result = acc.add(5).multiply(2).add(10).get(); // 5*2+10 = 20
    // acc.add(5) возвращает ссылку на acc, затем acc.multiply(2) и т.д.
}
```

Проверка на самоприсваивание в операторе присваивания
```
class String {
    char* data;
public:
    String& operator=(const String& other) {
        if (this == &other)    // проверка на самоприсваивание
            return *this;
        // освободить старую память, скопировать данные из other...
        return *this;
    }
};
```

Явное обращение к членам класса
```
void SomeClass::func(int x) {
    this->x = x;   // понятно, что this->x — поле, x — параметр
}
```

Передача указателя на себя в другую функцию
```
class Node {
    Node* next;
public:
    void attachTo(Node*& listHead) {
        next = listHead;
        listHead = this;    // вставляем себя в начало списка
    }
};
```
