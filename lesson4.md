# Правило пяти в C++

## Проблема управления ресурсами

C++ предоставляет программисту прямой контроль над памятью и другими ресурсами (файлы, сокеты, мьютексы и т.д.). \
Если класс владеет ресурсом (например, выделенной динамической памятью), он должен отвечать за его освобождение. \
Без специальных механизмов копирование таких объектов может привести к ошибкам: двойному освобождению, утечкам памяти, неопределённому поведению.

## Правило трёх

До C++11, если класс управлял ресурсом, который требовал нетривиального уничтожения, обычно требовалось определить три специальные функции-члена:
- Деструктор (~T()) – освобождает ресурс.
- Конструктор копирования (T(const T&)) – создаёт копию ресурса (глубокое копирование).
- Оператор присваивания копированием (T& operator=(const T&)) – освобождает старый ресурс и копирует новый.

Почему это правило возникает? \
Если компилятор генерирует эти функции автоматически, он выполняет почленное копирование полей. \
Для указателей это означает просто копирование адреса, а не копирование данных. \
В результате два объекта будут указывать на одну и ту же память, и при уничтожении одного из них произойдёт \
двойное освобождение (если оба вызовут delete), либо утечка, если память не будет освобождена вовсе.

## Правило пяти
---
С появлением семантики перемещения в C++11 добавились ещё две специальные функции: 
- Конструктор перемещения (T(T&&)) – перехватывает ресурс у временного объекта.
- Оператор присваивания перемещением (T& operator=(T&&)) – перемещает ресурс из одного объекта в другой.
---
**Таким образом мы должны получить:**
- Деструктор – освобождает ресурс.
- Конструктор копирования – создаёт копию ресурса (глубокое копирование).
- Оператор присваивания копированием – освобождает старый ресурс и копирует новый.
- Конструктор перемещения – перехватывает ресурс у временного объекта (перемещение).
- Оператор присваивания перемещением – перемещает ресурс из одного объекта в другой.

Если не определить эти функции явно, компилятор может сгенерировать их автоматически, но для классов, \
владеющих ресурсами, это почти всегда приводит к ошибкам (утечки, двойное освобождение). \
Цель правила пяти — обеспечить корректное и эффективное управление ресурсами. \
Правило пяти гласит: если класс управляет ресурсом, то обычно требуется определить все пять (или явно запретить некоторые из них). \
Часто при этом реализуют копирующие операции через глубокое копирование, а перемещающие – через перехват ресурса (обнуление источника).

### Конструктор копирования

Создаёт новый объект как копию существующего. Должен обеспечивать глубокое копирование всех ресурсов, чтобы новый объект владел собственной копией данных. \
Особенности реализации:
- Сигнатура: T(const T& other) или T(T& other) (последнее редко).
- Для глубокого копирования: выделить новую память, скопировать данные из other.
- Если класс содержит поля, которые сами управляют ресурсами (например, std::string), можно использовать их копирующие конструкторы.
- Важно обеспечить строгую гарантию безопасности исключений: если в процессе копирования произойдёт исключение, исходный объект не должен быть повреждён.
  
Пример:
```
Buffer(const Buffer& other) : size(other.size), data(new int[other.size]) {
    std::copy(other.data, other.data + size, data);
}
```
### Оператор присваивания копированием

Присваивает значение существующему объекту из другого объекта. \
Должен корректно освобождать старый ресурс и копировать новый, а также обрабатывать самоприсваивание. \
Особенности реализации:
- Сигнатура: T& operator=(const T& other).
- Должен возвращать ссылку на *this (для цепочек присваивания).
- Важно защититься от самоприсваивания (if (this == &other) return *this;).
- освободить старый ресурс, затем выделить новый и скопировать.

Пример:

```
Buffer& operator=(const Buffer& other) {
    if (this == &other) return *this;
    delete[] data;                     // освобождаем старый
    size = other.size;
    data = new int[size];
    std::copy(other.data, other.data + size, data); // механизм копировавния
    return *this;
}
```
### Конструктор перемещения
Создаёт новый объект, перехватывая ресурсы из временного (rvalue) объекта. Должен быть быстрым и не выбрасывать исключений (по возможности). \

Особенности реализации:
- Сигнатура: T(T&& other) noexcept – принимает rvalue-ссылку.
- После перемещения исходный объект остаётся в валидном, но неопределённом состоянии.
  Обычно обнуляют указатели, сбрасывают размеры, чтобы деструктор источника не освободил переданный ресурс.
- Перемещающие операции должны быть noexcept, если это возможно, чтобы контейнеры (например, std::vector) \
  могли использовать их при реаллокации без копирования.
```
Buffer(Buffer&& other) noexcept
    : data(other.data), size(other.size) {
    other.data = nullptr;
    other.size = 0;
}
```

### Оператор присваивания перемещением

Присваивает существующему объекту ресурсы из временного объекта. Должен освободить свои старые ресурсы и перехватить новые, \
оставив источник в валидном пустом состоянии. \
Особенности реализации:
- Сигнатура: T& operator=(T&& other) noexcept.
- Возвращает *this.
- Защита от самоприсваивания не столь критична, но полезна (хотя для rvalue ссылки самоприсваивание маловероятно).
- Как правило, реализуется либо через обмен с временным объектом (например, swap), либо аналогично перемещающему конструктору.
```
Buffer& operator=(Buffer&& other) noexcept {
    if (this == &other) return *this;
    delete[] data;
    data = other.data;
    size = other.size;
    other.data = nullptr;
    other.size = 0;
    return *this;
}
```

### Пример
```
#include <iostream>
#include <cstring>

class String {
    char* data;
    size_t len;

public:
    // Конструктор
    String(const char* str = "");

    // Деструктор
    ~String();

    // Конструктор копирования (глубокое копирование)
    String(const String& other);

    // Оператор присваивания копированием (copy assignment)
    String& operator=(const String& other);

    // Конструктор перемещения
    String(String&& other) noexcept;

    // Оператор присваивания перемещением
    String& operator=(String&& other) noexcept ;
    
};

int main() {
    String s1("Hello");
    String s2 = s1;           // копирование
    String s3("World");
    s3 = s1;                  // копирующее присваивание
    String s4 = std::move(s1); // перемещение (s1 становится пустым)
    s3 = std::move(s4);       // перемещающее присваивание
    // s1, s2, s3, s4 уничтожатся автоматически
}
```
## Правило нуля

Современный C++ рекомендует следовать правилу нуля: не писать собственные деструкторы, конструкторы копирования/перемещения и операторы присваивания, \
если класс не управляет ресурсами напрямую. Вместо этого используйте готовые обёртки, которые уже реализуют правильное поведение: \
std::vector, std::string, std::unique_ptr, std::shared_ptr и т.д. \
Правило нуля и правило пяти не противоречат друг другу: если вы вынуждены управлять ресурсом (например, из-за интерфейса на C), реализуйте все пять. \
Если же можно обойтись без этого – не пишите их вручную.
