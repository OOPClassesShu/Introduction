# Перегрузка операторов в C++


## Что такое перегрузка операторов и зачем она нужна
В C++ операторы (такие как +, -, *, ==, [] и другие) изначально определены только для встроенных типов (int, double, указатели и т.д.). \
Перегрузка операторов позволяет задать поведение этих операторов для пользовательских типов (классов и структур). \
Это даёт возможность работать с объектами так же естественно, как со встроенными типами, повышая выразительность и читаемость кода. 

Цели перегрузки:

- Интуитивно понятный интерфейс (например, сложение комплексных чисел через +).
- Единообразие: операции над объектами выглядят как над обычными числами.
- Возможность интеграции с шаблонами и стандартной библиотекой (контейнеры, алгоритмы часто требуют определённых операторов).

Важно помнить: перегрузка не меняет приоритет операторов, их ассоциативность или количество операндов. Нельзя создать новый оператор (например, **). \
Также не стоит нарушать ожидаемую семантику — перегруженный оператор должен делать то, что логично следует из его названия.

## Способы перегрузки операторов

Существует два основных способа перегрузить оператор:

- Как член класса – оператор становится методом класса. Левый операнд (для бинарных операторов) — это сам объект (this),
  правый передаётся аргументом. Унарные операторы не принимают аргументов (кроме постфиксных, о них позже).

- Как дружественная (или обычная) функция – оператор реализуется вне класса, обычно с доступом к приватным полям через дружественность.
  Это необходимо, когда левый операнд не является объектом вашего класса (например, операторы ввода/вывода <<, >>).

Пример: перегрузка + как метода и как функции
```
class Complex {
    double re, im;
public:
    Complex(double r = 0, double i = 0) : re(r), im(i) {}

    // Член класса (бинарный оператор)
    Complex operator+(const Complex& other) const {
        return Complex(re + other.re, im + other.im);
    }

    // Дружественная функция (для симметрии с double)
    friend Complex operator+(double lhs, const Complex& rhs);
};

// Вне класса
Complex operator+(double lhs, const Complex& rhs) {
    return Complex(lhs + rhs.re, rhs.im);
}
```

Рекомендация:

- Для операторов, изменяющих объект (+=, ++ и т.д.), часто используют члены класса.
- Для симметричных бинарных операторов (+, -, *, ==) лучше определять свободные функции.

## Какие операторы можно перегрузить

Перегрузить можно большинство операторов C++. Разделим их на группы:

**Арифметические и побитовые** 

- Унарные: +, -, ~, !, &, * (разыменование)
- Бинарные: +, -, *, /, %, &, |, ^, <<, >>

**Операторы сравнения** 

==, !=, <, >, <=, >= (обычно их определяют парами)

**Операторы присваивания** 
=, +=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=

**Инкремент и декремент** 

++ (префиксный и постфиксный), --

**Операторы доступа и вызова** 

[] (индексация), () (вызов функции), -> (доступ по указателю), ->*, .* (доступ к члену по указателю – последние два перегружаются редко)

**Операторы управления памятью** 

new, new[], delete, delete[]

**Операторы преобразования типов** 

operator тип() (например, operator double() const)

**Операторы потокового ввода/вывода** 

<<, >> – обычно перегружаются как свободные функции.
