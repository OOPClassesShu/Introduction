# Отношения между классами

## Вложенные классы

Вложенный класс (nested class) — это класс, объявленный внутри другого класса. \
В C++ такая конструкция является мощным инструментом для организации кода, инкапсуляции и сокрытия деталей реализации. \
На первый взгляд может показаться, что это просто синтаксический сахар, но на практике вложенные классы решают целый ряд архитектурных задач.

**Зачем нужны вложенные классы**

- Логическая группировка – класс, который нужен только внутри другого класса, помещается внутрь него. Это улучшает читаемость и показывает намерение.
- инкапсуляция и сокрытие – вложенный класс может быть объявлен в private или protected секции, скрывая его от внешнего мира.
  Пользователи внешнего класса даже не знают о существовании вложенного.
- Сужение области видимости – имя вложенного класса не загрязняет глобальное пространство имён или пространство имён внешнего класса.
- Доступ к private членам – вложенный класс имеет доступ ко всем private членам внешнего класса, если он объявлен в секции public, protected или private – сам факт вложенности даёт доступ.
- Реализация паттернов – например, итераторы, вспомогательные узлы списков, фабрики, классы-строители (Builder).
```
class Outer {
private:
    int x;
    static int s;

public:
    class Nested {          // вложенный класс
    public:
        void foo(Outer& o) {
            o.x = 42;       // доступ к private нестатическому члену через объект
            s = 100;        // доступ к private статическому члену
        }
    };
};

int Outer::s = 0;
```
 Nested находится внутри области видимости Outer. Снаружи обращаться к нему можно только как Outer::Nested. \
 
 Доступ к членам внешнего класса:

- К статическим членам можно обращаться напрямую (по имени или через Outer::).
- К нестатическим членам – только через объект внешнего класса (указатель, ссылку или сам объект). Вложенный класс не получает автоматически this внешнего класса.

Вложенный класс не имеет особого доступа к внешнему объекту
```
class Outer {
    int a;
public:
    class Inner {
    public:
        void func(Outer& o) { o.a = 5; }  // OK, через ссылку
        void func2() { a = 5; }           // Ошибка! Нет объекта Outer
    };
};
```

Вложенный класс и дружественность

- Вложенный класс автоматически является другом внешнего? Нет. Он просто имеет доступ к private/protected членам внешнего класса. Стандарт говорит: вложенный класс является членом внешнего, а члены класса имеют доступ к закрытым частям класса.
- Внешний класс не имеет автоматического доступа к private членам вложенного класса. Если нужен двусторонний доступ – используйте friend.

```
class Outer {
    class Inner {
        int secret;
    public:
        friend class Outer;  // теперь Outer может видеть secret
    };
};
```
 
Рекомендации
- Не переусердствуйте: глубокое вложение (класс в классе в классе) ухудшает читаемость.
- Если вложенный класс используется только внутри одного метода – возможно, лучше локальный класс (определённый внутри функции). Но локальные классы не могут быть шаблонами и имеют ограничения.
- Вложенный класс удобен, когда он является неотъемлемой частью реализации внешнего и не нужен в другой области.
- Помните, что вложенный класс не может обращаться к нестатическим членам внешнего без объекта – это частая ошибка.


## Агрегация

Агрегация – слабое отношение «часть–целое». Части могут существовать независимо от целого и даже быть общими для нескольких целых объектов. \
Целое не отвечает за создание и уничтожение частей, а только ссылается на них.

Характеристики:

- Части передаются в целое извне (через конструктор или метод).
- Целое не управляет временем жизни частей (не удаляет их в деструкторе).
- Часть может быть переиспользована в другом агрегаторе.
- Обычно используются сырые указатели, ссылки или std::shared_ptr (если нужно разделяемое владение).

Пример: университет и студенты \
Студент может учиться в университете, но при закрытии университета студент не исчезает – он переходит в другой вуз. \
Студент существует независимо от конкретного университета. Более того, один студент может посещать несколько кружков или даже числиться в двух учебных заведениях.
```
#include <iostream>
#include <vector>
#include <memory>

class Student {
    std::string name;
public:
    Student(const std::string& n) : name(n) {}
    void introduce() const { std::cout << "Student " << name << "\n"; }
};

class University {
    std::vector<Student*> students;   // агрегация: сырые указатели
public:
    void addStudent(Student* s) {
        students.push_back(s);
    }
    void showStudents() const {
        for (auto* s : students) s->introduce();
    }
    // Деструктор University НЕ удаляет студентов
};

int main() {
    Student alice("Alice");
    Student bob("Bob");
    // Студенты созданы независимо от университета

    University uni;
    uni.addStudent(&alice);
    uni.addStudent(&bob);

    uni.showStudents();
    // Университет уничтожается, но студенты (alice, bob) остаются живы
}
```

## Композиция

Композиция – это отношение, при котором «целое» полностью владеет своими «частями». Части не могут существовать отдельно от целого и уничтожаются вместе с ним.

Характеристики:
- Время жизни части полностью управляется целым.
- Часть создаётся внутри конструктора целого (или прямо в поле).
- Часть удаляется деструктором целого.
- Внешний код не имеет доступа к созданию или удалению частей по отдельности.
- При копировании целого обычно копируются и части (глубокое копирование).

Пример: дом и комнаты
Комнаты не существуют без дома. Если дом сносят, все комнаты исчезают. Невозможно «переставить» комнату из одного дома в другой без разрушения стен.

```
#include <iostream>
#include <string>
#include <vector>

class Room {
    std::string name;
public:
    Room(const std::string& n) : name(n) {
        std::cout << "Room " << name << " created\n";
    }
    ~Room() {
        std::cout << "Room " << name << " destroyed\n";
    }
    void describe() const { std::cout << "Room: " << name << "\n"; }
};

class House {
    std::vector<Room> rooms;   // композиция: объекты Room хранятся по значению
public:
    House() {
        rooms.emplace_back("Living room");
        rooms.emplace_back("Kitchen");
        rooms.emplace_back("Bedroom");
        std::cout << "House built.\n";
    }
    // Деструктор автоматически уничтожит все Room
    void showRooms() const {
        for (const auto& r : rooms) r.describe();
    }
};

int main() {
    House myHouse;
    myHouse.showRooms();
    // При выходе из блока House уничтожится, а с ним и все комнаты
}
```

## Наследование

Наследование – это механизм, позволяющий создать новый класс (производный, дочерний, подкласс) на основе уже существующего \
(базового, родительского, суперкласса). Производный класс автоматически получает все члены базового \
(кроме конструкторов, деструктора, операторов присваивания и друзей) и может добавлять свои.
```
class Base {
    // члены
};

class Derived : [доступ] Base {
    // дополнительные члены
};
```

Типы наследования и доступ к членам

| Наследование \ Члены в базовом классе |	public |	protected	| private |
|---------------------------------------|--------|------------|---------|
| public |	public |	protected	 | недоступен |
| protected |	protected |	protected	| недоступен |
| private |	private |	private | недоступен |

- public наследование – сохраняет уровни доступа.
- protected наследование – все унаследованные члены становятся protected в производном классе (кроме private).
- private наследование – все унаследованные члены становятся private.

Рекомендация: на лекции мы будем использовать только public наследование.

Конструкторы при наследовании

Важнейший момент: конструкторы базового класса не наследуются. Производный класс обязан вызвать конструктор базового класса в своей инициализирующей секции.

Порядок вызова конструкторов:

- Сначала вызывается конструктор базового класса.
- Затем – конструкторы полей-членов (в порядке объявления).
- И наконец – тело конструктора производного класса.

Пример:
```
#include <iostream>

class Base {
public:
    Base(int x) {
        std::cout << "Base constructor, x = " << x << "\n";
    }
    ~Base() {
        std::cout << "Base destructor\n";
    }
};

class Derived : public Base {
    int a;
public:
    // Явный вызов конструктора Base
    Derived(int x, int y) : Base(x), a(y) {
        std::cout << "Derived constructor, a = " << a << "\n";
    }
    ~Derived() {
        std::cout << "Derived destructor\n";
    }
};

int main() {
    Derived d(10, 20);
    return 0;
}
```
