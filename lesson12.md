# Умные указатели

RAII (Resource Acquisition Is Initialization) – ключевая идиома C++ «захват ресурса должен быть инициализацией объекта». \
ресурс (память, файл, мьютекс) захватывается в конструкторе и освобождается в деструкторе. Умные указатели – это RAII‑обёртки над сырыми указателями.\

## Умный указатель

Шаблон класса, который:
- хранит сырой указатель;
- перегружает операторы * и -> для доступа к объекту;
- автоматически вызывает delete (или другой удалитель) в своём деструкторе.

```
template<typename T>
class SimpleSmartPtr {
    T* ptr;
public:
    explicit SimpleSmartPtr(T* p = nullptr) : ptr(p) {}
    ~SimpleSmartPtr() { delete ptr; }
    T& operator*() const { return *ptr; }
    T* operator->() const { return ptr; }
};
```
Стандартная библиотека предоставляет три основных вида: unique_ptr, shared_ptr, weak_ptr.

### std::unique_ptr – эксклюзивное владение

- Нельзя копировать, только перемещать.
- В любой момент времени только один unique_ptr владеет объектом.
- При разрушении unique_ptr уничтожает объект.
- Нулевые накладные расходы по сравнению с сырым указателем (в типичной реализации).

Создание и базовое использование:
```
#include <memory>

std::unique_ptr<int> u1 = std::make_unique<int>(10);
auto u2 = std::make_unique<std::string>("Hello");

// Доступ к объекту
std::cout << *u1 << ", " << u2->size() << '\n';

// Перемещение – передача владения
std::unique_ptr<int> u3 = std::move(u1); // теперь u1 пуст
```

Использование в контейнерах и функциях:
```
std::vector<std::unique_ptr<int>> vec;
vec.push_back(std::make_unique<int>(5));
vec.push_back(std::make_unique<int>(6));
// автоматическое удаление при разрушении вектора

void take_ownership(std::unique_ptr<int> p) { /* ... */ }
void observe(const std::unique_ptr<int>& p) { /* ... */ } // не берёт владение

auto p = std::make_unique<int>(42);
observe(p);                // ok
take_ownership(std::move(p)); // передаём владение
// p теперь пуст
```

### std::shared_ptr – разделяемое владение

Несколько shared_ptr могут владеть одним объектом.

- Внутри ведётся счётчик ссылок (control block).
- Объект удаляется, когда счётчик становится 0.
- Накладные расходы: два указателя (на объект и на блок управления) плюс атомарные операции для счётчика.

Создание и использование:
```
auto sp1 = std::make_shared<int>(100); // предпочтительный способ
std::shared_ptr<int> sp2(sp1);         // копирование, счётчик = 2
std::shared_ptr<int> sp3 = sp1;        // теперь счётчик = 3

std::cout << *sp1 << ", use_count: " << sp1.use_count() << '\n';
// sp1, sp2, sp3 выходят из области видимости – объект удаляется
```

Передача в функции:
```
void process(std::shared_ptr<int> p) { /* разделяемое владение */ }
void observe(const std::shared_ptr<int>& p) { /* только чтение */ }

auto sp = std::make_shared<int>(42);
process(sp);   // копируем – счётчик увеличивается
observe(sp);   // ссылка – счётчик не меняется
```

### Циклические ссылки и std::weak_ptr

Проблема: два shared_ptr ссылаются друг на друга → счётчики никогда не станут 0 → утечка.
```
struct Node {
    std::shared_ptr<Node> next;
    ~Node() { std::cout << "deleted\n"; }
};

auto a = std::make_shared<Node>();
auto b = std::make_shared<Node>();
a->next = b;  // a владеет b
b->next = a;  // b владеет a → цикл, утечка
```
Решение – weak_ptr:

- Не владеет объектом, не увеличивает счётчик ссылок.
- Нельзя разыменовать напрямую – сначала нужно получить shared_ptr через lock() (если объект ещё жив).

```
struct Node {
    std::weak_ptr<Node> next;   // слабая ссылка, цикла нет
};

auto a = std::make_shared<Node>();
auto b = std::make_shared<Node>();
a->next = b;
b->next = a;  // теперь цикл не образуется

// получение shared_ptr из weak_ptr
if (auto locked = a->next.lock()) {
    // используем locked
} else {
    // объект уже удалён
}
```

## Рекомендации

- Предпочитайте unique_ptr везде, где возможно. Он легче и яснее выражает владение.
  Только когда владение действительно разделяется, используйте shared_ptr.
- Всегда используйте make_unique и make_shared (кроме случаев, когда нужен пользовательский удалитель или weak_ptr
  вместе с shared_ptr для больших объектов – но это нюансы).
- Не используйте get() без крайней необходимости – он возвращает сырой указатель, которым можно случайно удалить объект
  или вызвать delete. Только для совместимости со старым кодом.
```
auto sp = std::make_shared<int>(5);
int* raw = sp.get();          // ok, но осторожно
delete raw;                   // ошибка: двойное удаление!
```
Передача умных указателей в функции:

- Если функция не сохраняет указатель и не меняет владение → передавайте сырой указатель или ссылку: void f(T* ptr) или void f(T& ref).
- Если функция берёт владение → void f(std::unique_ptr<T> p).
- Если функция разделяет владение → void f(std::shared_ptr<T> p).
- Если функция только наблюдает за shared_ptr без продления жизни → const std::shared_ptr<T>&.

Возврат умных указателей из функций:
- Возвращайте unique_ptr – это естественный способ передать владение вызывающему.
- Возвращайте shared_ptr только если действительно нужно разделяемое владение.
