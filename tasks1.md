# Практические задачки 1

Задание 1. 

Класс «Счётчик» 

Реализуйте класс Counter, который хранит целое неотрицательное значение. 
Класс должен содержать:
- Приватное поле value (тип int).
- Конструктор по умолчанию, инициализирующий значение нулём.
- Конструктор с параметром, инициализирующий значение переданным числом (если переданное число отрицательное, устанавливать значение в 0).
- Метод get() для получения текущего значения.
- Метод set(int newValue) для установки нового значения (с проверкой: если newValue < 0, оставить старое значение или установить 0 – на ваше усмотрение).
- Методы increment() и decrement() — увеличивают и уменьшают счётчик на 1. При попытке уменьшить значение ниже 0, счётчик должен остаться 0 (или можно просто не менять).
- Деструктор, который выводит сообщение "Counter destroyed" (для наблюдения за временем жизни).
- статическое поле totalCount, которое увеличивается при создании каждого нового объекта и уменьшается при уничтожении, и статический метод getTotalCount(), возвращающий это число.

В функции main() продемонстрируйте работу всех методов: создайте объекты разными способами (по умолчанию, с параметром), измените значения, выведите на экран. 
Покажите, что деструктор вызывается для объектов автоматически.


Задание 2. 

Класс «Logger» с цепочкой вызовов и статическими членами


Реализуйте класс Logger, который накапливает сообщения и умеет выводить их. 
Класс должен:

- Иметь статическое поле logLevel (уровень логирования: 0 — ошибки, 1 — предупреждения, 2 — информация) и статический метод setLogLevel(int).
- Для каждого экземпляра логгера можно добавлять сообщения с определённым уровнем.
  Методы error(const char*), warning(const char*), info(const char*) добавляют сообщение в буфер только если его уровень не выше установленного глобального уровня.
  Эти методы должны возвращать ссылку на текущий объект, чтобы можно было строить цепочки: logger.error("Error1").warning("Warning").info("Info").
- Метод flush() выводит все накопленные сообщения (например, в std::cout по одному в строке) и очищает буфер. Возвращает ссылку на себя.

В main() продемонстрируйте:
- Установку уровня логирования через статический метод.
- Создание логгера, добавление нескольких сообщений с разными уровнями, затем flush().
- Покажите, что при низком уровне сообщения игнорируются.
- Используйте цепочку вызовов: logger.info("Start").warning("Check").error("Fail").flush();
