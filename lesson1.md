# Занятие 1

## Что такое ООП?

Объектно-ориентированное программирование (ООП) — это методология программирования, основанная на представлении программы в виде совокупности взаимодействующих объектов, 
каждый из которых является экземпляром определённого класса. 

Основные принципы:
- **Инкапсуляция** — объединение данных и методов их обработки внутри класса, скрытие внутренней реализации.
- **Наследование** — возможность создания нового класса на основе существующего с добавлением новых свойств и методов.
- **Полиморфизм** — способность объектов с одинаковым интерфейсом иметь различную реализацию.

## Классы и объекты

- Класс — пользовательский тип данных, который объединяет данные (поля) и функции (методы) для работы с ними.
- Объект — конкретный экземпляр класса.

Пример объявления класса:
```
class Point {
public:
    double x, y; // поля класса
    void print() { // метод (функции-члены класса)
        std::cout << "(" << x << ", " << y << ")" << std::endl;
    }
};
```
Создание объекта:
```
  Point p;
  p.x = 3.5;
  p.y = 2.1;
  p.print();
```

## Инкапсуляция и модификаторы доступа

Модификаторы доступа определяют, кто может обращаться к членам класса:

- **public** — доступны всем.
- **private** — доступны только внутри класса.
- **protected** — доступны внутри класса и в классах-наследниках.

Инкапсуляция достигается сокрытием данных (private) и предоставлением публичных методов для доступа к ним (геттеры/сеттеры).
```
class Rectangle {
private:
    double width, height;
public:
    void setDimensions(double w, double h) {
        if (w > 0 && h > 0) {
            width = w;
            height = h;
        }
    }
    double getArea() {
        return width * height;
    }
};
```
## Определение метода (функций -членов класса)
### Определение метода (функций -членов класса) внутри класса

Метод определяется непосредственно в теле класса. Такие методы автоматически считаются inline 
(если они не слишком сложные, компилятор может подставить их код в место вызова). 

Синтаксис:
```
class MyClass {
public:
    void method1() {
        // реализация метода
    }
    
    int method2() {
        return 42;
    }
};
```

**Преимущества:**

- Удобно для коротких методов (например, геттеров/сеттеров).
- Код интерфейса и реализации находится в одном месте.
- Компилятору легче выполнить встраивание (inlining).

**Недостатки:**

- Если метод длинный, определение внутри класса загромождает интерфейс и снижает читаемость.
- Любое изменение реализации требует перекомпиляции всех файлов, которые включают этот заголовочный файл (так как реализация видна везде).

### Определение метода вне класса

Метод объявляется внутри класса, а определяется за его пределами с помощью оператора разрешения области видимости ::. 
При этом можно использовать ключевое слово inline, если требуется явное встраивание. 

Синтаксис:
```
// Заголовочный файл (MyClass.h)
class MyClass {
public:
    void method1();        // только объявление
    int method2();
};

// Файл реализации (MyClass.cpp)
#include "MyClass.h"

void MyClass::method1() {
    // реализация метода
}

int MyClass::method2() {
    return 42;
}
```

**Преимущества:**

- Чёткое разделение интерфейса (заголовочный файл) и реализации (файл .cpp).
- Изменение реализации не требует перекомпиляции кода, который использует класс (достаточно перелинковать объектный файл).
- Класс выглядит более компактно и легче воспринимается.

**Недостатки:**

- Для коротких методов может быть менее удобно.
- При желании сделать метод встраиваемым нужно явно указывать inline (как при определении, так и в объявлении),
  либо поместить определение в заголовочный файл (но вне класса), чтобы оно было видно во всех единицах трансляции.

**Размещение определений в разных файлах**
Обычно определения вне класса помещают в файлы .cpp, а объявления – в заголовочные файлы. 
Это позволяет избежать дублирования кода при множественных включениях заголовка и ускоряет компиляцию. 

**Немного вперед.** 
 
**Шаблонные классы** 

Для шаблонных классов определение методов обязано находиться в заголовочном файле (либо в одном файле с объявлением), 
так как компилятор должен видеть полный код при инстанцировании шаблона. 
При этом можно определять методы внутри класса или вне его, но в том же заголовочном файле. 

**Константные методы и спецификаторы** 

При определении метода вне класса необходимо повторить все спецификаторы (например, const, override, final, спецификатор исключений), 
кроме модификатора доступа (он задаётся только в объявлении). 

### Пример с комментариями
```
// point.h
#pragma once

class Point {
private:
    double x, y;
public:
    Point(double x = 0, double y = 0);   // объявление конструктора
    void setX(double value);              // объявление
    double getX() const;                  // объявление
    double distanceTo(const Point& other) const; // объявление
};

// point.cpp
#include "point.h"
#include <cmath>

Point::Point(double x, double y) : x(x), y(y) {}  // определение конструктора

void Point::setX(double value) {   // определение вне класса
    x = value;
}

double Point::getX() const {       // определение вне класса
    return x;
}

double Point::distanceTo(const Point& other) const {
    double dx = x - other.x;
    double dy = y - other.y;
    return std::sqrt(dx*dx + dy*dy);
}
```

## Геттеры и сеттеры

Геттер (getter) — это метод класса, который позволяет получить значение закрытого (private) поля. 
Сеттер (setter) — метод, который позволяет изменить значение закрытого поля, обычно с контролем допустимости новых данных. 
Они являются основным инструментом реализации инкапсуляции — сокрытия внутреннего состояния объекта и предоставления контролируемого доступа к нему. 
```
class Person {
private:
    std::string name;
    int age;
public:
    // Getter для name
    std::string getName() const {
        return name;
    }
    // Setter для name
    void setName(const std::string& newName) {
        name = newName;
    }

    // Getter для age
    int getAge() const {
        return age;
    }
    // Setter для age
    void setAge(int newAge) {
        if (newAge >= 0 && newAge < 150)  // простая валидация
            age = newAge;
    }
};

Person p;
p.setName("Alice");
p.setAge(30);
std::cout << p.getName() << " is " << p.getAge() << " years old.\n";
```

## Зачем нужны геттеры и сеттеры

- Контроль доступа. Можно запретить изменение поля (только getter), разрешить только чтение. Или разрешить изменение только определённым способом.
- Валидация. Сеттер может проверять корректность нового значения перед присваиванием, защищая объект от недопустимого состояния.
- Вычисляемые значения. Геттер может не просто возвращать поле, а вычислять значение на основе других полей. Например, площадь прямоугольника по ширине и высоте.
- Побочные эффекты.Сеттер может обновлять связанные поля, уведомлять наблюдателей, логировать изменения и т.д.
- Сохранение инвариантов. Класс может гарантировать, что его внутреннее состояние всегда корректно (инвариант). При прямом доступе к полям инвариант легко нарушить.
- Абстракция и гибкость. Можно изменить внутреннее представление данных (например, хранить температуру в Цельсиях, а геттер/сеттер работать с Фаренгейтами),
  не меняя интерфейс класса.
- Отладка и логирование. В геттер/сеттер можно добавить вывод отладочной информации, точки останова.
- Совместимость с интерфейсами. Если класс участвует в полиморфных иерархиях, виртуальные геттеры/сеттеры позволяют настраивать поведение в производных классах.
