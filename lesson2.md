# Конструкторы

## Что такое конструктор?

**Конструктор** — это специальная функция-член класса, которая автоматически вызывается при создании объекта. 
Его основная задача — инициализировать объект, привести его в корректное начальное состояние (выделить ресурсы, установить начальные значения полей и т.д.). 

- Имя конструктора совпадает с именем класса.
- Не имеет возвращаемого типа (даже void).
- Может быть перегружен (класс может иметь несколько конструкторов с разными параметрами).
- Может быть объявлен в секциях public, protected или private (в зависимости от требуемого уровня доступа).
- Не может быть виртуальным (хотя деструктор может), но конструкторы могут вызывать виртуальные функции.

## Синтаксис объявления и определения
```
class MyClass {
public:
    // Объявление конструкторов
    MyClass();                     // по умолчанию
    MyClass(int param);            // с параметром
};
```
Определение конструктора может быть внутри класса или вне его:
```
// Внутри класса
MyClass() { /* ... */ }

// Вне класса
MyClass::MyClass() { /* ... */ }
```
## Виды конструкторов
### Конструктор по умолчанию (default constructor)

Конструктор, который можно вызвать без аргументов. Если не определить ни одного конструктора, компилятор автоматически сгенерирует публичный конструктор по умолчанию, 
который выполняет default-initialization (для встроенных типов оставляет значения неопределёнными, для классовых типов вызывает их конструкторы по умолчанию). 
```
class A {
public:
    int x;
    std::string s;
    // Если других конструкторов нет, компилятор создаст A() {}
};

A a; // ok
```

Если определён любой другой конструктор, конструктор по умолчанию автоматически не генерируется

```
class B {
public:
    B(int) {}  // пользовательский конструктор
    // B() не генерируется!
};

// B b; // ошибка: нет конструктора по умолчанию
```

### Конструктор с параметрами

Позволяет инициализировать объект переданными значениями.
```
class Point {
    double x, y;
public:
    Point(double x_val, double y_val) : x(x_val), y(y_val) {}
};

Point p(3.0, 4.0);  // традиционный синтаксис
Point p2 = {3.0, 4.0}; // uniform initialization (C++11)
```
### Другие конструкторы, которые мы будем рассматривать чуть позже

- Конструктор копирования (copy constructor)
- Конструктор перемещения (move constructor) (C++11)
- Конструктор от std::initializer_list (C++11)
 

## Перегрузка конструкторов

Класс может иметь несколько конструкторов с разными наборами параметров. Компилятор выбирает подходящий по правилам разрешения перегрузки.
```
class Rectangle {
    double width, height;
public:
    Rectangle() : width(1), height(1) {}                 // по умолчанию
    Rectangle(double side) : width(side), height(side) {} // квадрат
    Rectangle(double w, double h) : width(w), height(h) {} // прямоугольник
};

Rectangle r1;        // default
Rectangle r2(5.0);   // квадрат
Rectangle r3(4, 6);  // прямоугольник
```
## Инициализация по умолчанию внутри класса (in-class member initializers)

Можно задать значения по умолчанию прямо при объявлении поля. Тогда конструктор может не инициализировать его явно.
```
class Widget {
    int id = 0;                // значение по умолчанию
    std::string name = "anon";
    double weight = 1.0;
public:
    Widget() = default;        // использует значения по умолчанию
    Widget(int i) : id(i) {}   // id переопределяется, остальные по умолчанию
};
```

## Инициализация полей класса (member initializer list)

Самый эффективный и предпочтительный способ инициализации полей (особенно констант, ссылок и полей классовых типов, не имеющих конструктора по умолчанию). 
Выполняется до входа в тело конструктора. 
```
class Example {
    const int c;
    int& ref;
    std::string s;
public:
    Example(int x, int& r) : c(x), ref(r), s("hello") {
        // тело конструктора
    }
};
```
**ВАЖНО** Поля инициализируются в порядке их объявления в классе, независимо от порядка в списке инициализации.

## Инициализация в теле конструктора (присваивание)

Можно присвоить значения полям в теле, но это не инициализация, а присваивание (сначала поля были инициализированы по умолчанию, затем перезаписаны). 
Для сложных типов это может быть менее эффективно. 
```
class BadExample {
    std::string s;
public:
    BadExample(const char* str) {
        s = str;   // сначала s инициализирована пустой строкой, потом присвоено
    }
};

// Лучше: BadExample(const char* str) : s(str) {}
```

## Делегирование конструкторов (C++11)

Делегирование конструкторов (constructor delegation) — возможность, появившаяся в C++11, позволяющая одному конструктору класса вызывать 
другой конструктор того же класса из своего списка инициализации. Это позволяет избежать дублирования общего кода инициализации в нескольких конструкторах. 
До C++11, если у класса было несколько конструкторов, они часто дублировали инициализацию полей, либо приходилось выносить общий код в отдельную функцию init(), 
что было неэффективно (приходилось дважды инициализировать поля: сначала по умолчанию, затем в init). Делегирование решает эту проблему элегантно.

```
class Employee {
    std::string name;
    int id;
    double salary;
public:
    // Основной конструктор
    Employee(const std::string& n, int i, double s) 
        : name(n), id(i), salary(s) {}

    // Делегирующий конструктор (вызывает основной)
    Employee(const std::string& n) : Employee(n, 0, 0.0) {}

    // Ещё один делегирующий
    Employee() : Employee("Unknown") {}
};
```
Делегирование указывается в списке инициализации как вызов другого конструктора с соответствующими аргументами. 
После этого список инициализации не может содержать инициализацию других полей или базовых классов — делегирование должно быть единственным элементом списка. 
После вызова другого конструктора поля уже инициализированы, и в теле текущего конструктора можно выполнять дополнительные действия.

 ### Цепочки делегирования

 Можно строить цепочки: конструктор A делегирует B, B делегирует C и т.д.
```
class Date {
    int day, month, year;
public:
    Date(int d, int m, int y) : day(d), month(m), year(y) {}
    Date(int d, int m) : Date(d, m, 2024) {}
    Date() : Date(1, 1) {}
};
```

### Порядок выполнения

Сначала полностью выполняется целевой конструктор (его список инициализации и тело). Затем выполняется тело делегирующего конструктора. 
Таким образом, целевой конструктор гарантированно завершает инициализацию объекта до того, как делегирующий конструктор начнёт выполнять свой код. 
Это важно для корректной работы с частично инициализированными объектами.

### Примеры
```
class Student {
    std::string name;
    int id;
    double gpa;
public:
    Student(const std::string& n, int i, double g) 
        : name(n), id(i), gpa(g) {
        validate();
    }

    Student(const std::string& n, int i) 
        : Student(n, i, 0.0) {}   // делегирование

    Student() 
        : Student("", 0, 0.0) {}  // делегирование
};
```

**Установка значений по умолчанию**

```
class Time {
    int hour, minute, second;
public:
    Time(int h, int m, int s) : hour(h), minute(m), second(s) {
        normalize(); // приводит время к корректному диапазону
    }

    Time(int h, int m) : Time(h, m, 0) {}
    Time(int h) : Time(h, 0, 0) {}
    Time() : Time(0, 0, 0) {}
};
```
**Валидация параметров**
Часто валидацию лучше проводить в целевом конструкторе. Делегирующие могут полагаться на то, что целевой выполнит проверки.
```
class Range {
    int low, high;
public:
    Range(int l, int h) : low(l), high(h) {
        if (l > h) throw std::invalid_argument("low must be <= high");
    }

    explicit Range(int h) : Range(0, h) {}   // диапазон от 0 до h
    Range() : Range(0, 10) {}                 // диапазон по умолчанию
};
```
**Дополнительная логика**
Если требуется особая логика в делегирующем конструкторе до вызова целевого, это невозможно — делегирование происходит в списке инициализации, 
т.е. до любого кода в теле. Но можно выполнить логику после делегирования:
```
class Widget {
    int id;
    std::string name;
public:
    Widget(int i, const std::string& n) : id(i), name(n) {
        std::cout << "Widget created: " << name << std::endl;
    }

    Widget(const std::string& n) : Widget(0, n) {
        // можно добавить дополнительную инициализацию уже после основного конструктора
        std::cout << "Additional setup for string-only ctor\n";
    }
};
```

# Деструктор

## Что такое деструктор?

Деструктор — это специальная функция-член класса, которая автоматически вызывается при уничтожении объекта 
(когда объект выходит из области видимости, удаляется через delete, или программа завершается). 
Его основная задача — освободить ресурсы, захваченные объектом за время его жизни (память, файловые дескрипторы, сокеты, мьютексы и т.д.), 
выполнить завершающие действия.

- Имя деструктора совпадает с именем класса, но с префиксом ~ (тильда).
- Не имеет параметров и не возвращает значения (даже void).
- Не может быть перегружен — у класса может быть только один деструктор.
- Может быть объявлен как virtual, protected, private.
- Вызывается автоматически, но его можно вызвать и явно (редко, обычно не нужно).

```
class MyClass {
public:
    ~MyClass();            // объявление деструктора
};

MyClass::~MyClass() {      // определение вне класса
    // код освобождения ресурсов
}
```
Можно определить деструктор прямо внутри класса:
```
class MyClass {
public:
    ~MyClass() {
        // тело деструктора
    }
};
```
## Когда вызывается деструктор?

- Для автоматических (локальных) объектов — при выходе из блока (например, закрывающая фигурная скобка).
- Для глобальных и статических объектов — при завершении программы (после main).
- Для объектов, созданных через new — при вызове delete.
- Для временных объектов — в конце выражения, в котором они созданы.
- Для элементов массива — при уничтожении массива (вызывается для каждого элемента).
- Для подобъектов (базовые классы, нестатические поля) — автоматически после тела деструктора производного/содержащего класса.

## Автоматически генерируемый деструктор

Если мы не объявляем деструктор, компилятор генерирует его автоматически (публичный, невиртуальный, с пустым телом). 
Сгенерированный деструктор вызывает деструкторы всех нестатических полей (в порядке, обратном их объявлению) и деструктор базового класса (если есть).

Однако автоматической генерации не происходит, если класс имеет объявленный пользователем деструктор, конструктор копирования, оператор присваивания 
или конструктор перемещения. 

## Когда нужно писать свой деструктор?

Правило трёх/пяти: Если класс управляет ресурсами (динамическая память, файлы, сокеты и т.п.), он должен определять:

- деструктор,
- конструктор копирования,
- оператор присваивания,
- (и, возможно, конструктор перемещения и оператор перемещения).

Если деструктор не определён, компилятор сгенерирует пустой, и ресурсы не освободятся (утечка). Примеры классов, которым нужен свой деструктор:

- Классы, содержащие указатели на память, выделенную через new.
- Классы, открывающие файлы (fopen, std::fstream требует закрытия, но обычно RAII-обёртки сами закрывают).
- Классы, работающие с системными ресурсами (мьютексы, сокеты, дескрипторы Windows).
- Контейнеры STL и умные указатели уже управляют ресурсами, поэтому в классах, которые их используют, свой деструктор обычно не нужен.

## Явный вызов деструктора

Деструктор можно вызвать явно через obj.~ClassName(). Это необходимо крайне редко, например:

- При использовании размещающего new (placement new) — нужно явно вызывать деструктор перед освобождением памяти.
- В некоторых низкоуровневых задачах управления памятью.

```
char buffer[sizeof(MyClass)];
MyClass* obj = new (buffer) MyClass;  // placement new
// ... 
obj->~MyClass();  // явный вызов деструктора
// память buffer освобождать не нужно (она на стеке)
```

## Способы создания экземпляра класса

### Point p1; – конструктор по умолчанию
```
Point p1;   // вызов конструктора по умолчанию
```
Если у класса Point есть конструктор по умолчанию (без параметров), то при таком объявлении он вызывается. 
Конструктор по умолчанию может быть:

- сгенерирован компилятором автоматически, если не объявлен ни один конструктор;
- определён пользователем;
- или явно запрошен = default.

**Важно:** Если конструктор по умолчанию отсутствует, а есть только конструкторы с параметрами, эта строка вызовет ошибку компиляции. 
Для встроенных типов (int, double и т.п.) они остаются неинициализированными, но для полей классовых типов вызываются их конструкторы по умолчанию.

### Прямая инициализация Point p2(10, 20);
```
Point p2(10, 20);   // вызов конструктора Point(int, int)
```
Компилятор ищет конструктор, который можно вызвать с переданными аргументами (здесь два целых числа). 
Если такой конструктор существует он вызывается. Это наиболее прямой и часто используемый способ инициализации.

### Агрегатная инициализация

```
Point p3 = {30, 40};   // агрегатная инициализация
```

Эта форма работает только если Point является агрегатом. Агрегат в C++ – это: 
массив, или класс (struct/class), у которого:
- нет пользовательских конструкторов (кроме = default или = delete);
- нет виртуальных функций;
- нет виртуальных базовых классов.
Для агрегатов инициализация списком { ... } инициализирует поля по порядку их объявления.
В примере p3.x = 30, p3.y = 40. Если список короче, чем количество полей, остальные поля инициализируются значениями по умолчанию
(для встроенных типов – нулём, для классовых – конструктором по умолчанию).

Пример агрегата:
```
struct Point { int x, y; };   // агрегат
Point p = {1, 2};              // x=1, y=2
```
### Явное создание
```
Point p4 = Point(50, 60);   // создание временного объекта и инициализация p4
```
Здесь происходит следующее: 

Создаётся временный объект типа Point с помощью конструктора Point(int, int). 
Этот временный объект используется для инициализации переменной p4 (вызывается конструктор копирования или перемещения). 
Временный объект уничтожается. 
Однако на практике компиляторы применяют оптимизацию пропуска копирования (copy elision). 
Начиная с C++17, в некоторых случаях это копирование гарантированно исключается (mandatory elision). 
Для данного примера с C++17 объект p4 будет создан непосредственно вызовом конструктора Point(50,60), без временного объекта и копирования. 
До C++17 компиляторы тоже обычно применяли RVO (Return Value Optimization) в таких ситуациях, но это не гарантировалось. 

### Uniform initialization с фигурными скобками
```
Point p5{1, 2};   // прямая список-инициализация
```
Если класс не агрегат, вызывает конструктор, подходящий под список (возможно, std::initializer_list конструктор). 
Если класс агрегат, работает как агрегатная инициализация. Этот способ предпочтителен в современном C++, так как он предотвращает 
сужающие преобразования (narrowing conversions) и единообразен.

### Копирующая инициализация с фигурными скобками
```
Point p6 = {1, 2};   // уже рассмотрели для агрегатов, но может работать и для классов с initializer_list
```

### Инициализация с помощью auto

```
auto p7 = Point{1, 2};
auto p8 = new Point{1, 2}; // динамический объект
```
