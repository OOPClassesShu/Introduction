# STL

**STL (Standard Template Library)** — это часть стандартной библиотеки C++, построенная на принципах обобщённого программирования. \
Она предоставляет готовые, эффективные и типобезопасные структуры данных и алгоритмы.

STL решает 
- Написание собственных коллекций (списков, векторов) для каждого проекта — долго, ошибкоопасно и неэффективно.
- Повторение типовых алгоритмов (поиск, сортировка) с разными типами данных.
- Смешение указателей, динамической памяти и ручного управления ресурсами.

Разделение данных и операций над ними. Данные хранятся в контейнерах, алгоритмы работают с ними через единый интерфейс — итераторы. \
Это позволяет одному алгоритму (например, sort) работать с вектором, списком или даже массивом.

STL строится на четырёх «китах»:

- Контейнеры — классы для хранения наборов объектов.
- Итераторы — обобщение указателей, позволяющее перебирать элементы контейнера.
- Алгоритмы — функции, выполняющие операции над последовательностями (поиск, сортировка, копирование и т.д.).
- Функторы (функциональные объекты) и адаптеры — механизмы настройки поведения алгоритмов.

Все компоненты STL — шаблонные.

Контейнеры

| Категория |	Примеры |	Особенности |
|-----------|---------|-------------|
|Последовательные|	vector, deque, list, array, forward_list |	Элементы упорядочены, порядок определяется пользователем.|
|Ассоциативные|	set, multiset, map, multimap |	Двоичное дерево поиска. Быстрый поиск O(log n). Ключи упорядочены.|
|Неупорядоченные| (хеш-таблицы, C++11)	unordered_set, unordered_map и их multi-версии |	Среднее O(1) поиск, но неупорядочены.|

## Итераторы

Итератор — объект, который «умеет»:

- передвигаться по элементам (++, --),
- получать доступ к элементу (*, ->),
- сравниваться (==, !=).

Категории итераторов (от простого к сложному):

- Input / Output — однопроходные.
- Forward — многоходовые вперёд.
- Random Access — произвольный доступ (vector, deque, array).

Пример
```
vector<int> vec = {10,20,30};
for (auto it = vec.begin(); it != vec.end(); ++it) {
    cout << *it << " ";
}
```
Диапазон [begin, end) — полуоткрытый, end указывает за последний элемент. Это универсальное соглашение STL.

## Алгоритмы

Более 100 алгоритмов в заголовке <algorithm>. Работают с итераторами, не зависят от конкретного контейнера.

```
#include <algorithm>
#include <vector>
#include <numeric>   // для accumulate

vector<int> data = {5,2,8,1,9};

// Сортировка
sort(data.begin(), data.end());          // {1,2,5,8,9}

// Поиск
auto it = find(data.begin(), data.end(), 5);
if (it != data.end()) cout << "Найден: " << *it;

// Копирование
vector<int> copy;
copy_if(data.begin(), data.end(), back_inserter(copy), [](int x){ return x > 4; });

// Сумма (из <numeric>)
int sum = accumulate(data.begin(), data.end(), 0);
```

Модифицирующие и немодифицирующие алгоритмы. Например, for_each, transform, remove_if.

## Функторы и лямбда-выражения

Алгоритмы часто позволяют передать пользовательское действие (предикат, операцию). Это можно сделать через:

- Указатель на функцию,
- Функтор (класс с перегруженным operator()),
- Лямбда-выражение.

```
sort(v.begin(), v.end(), [](int a, int b) { return a > b; }); // по убыванию

struct Greater {
    bool operator()(int a, int b) const { return a > b; }
};
sort(v.begin(), v.end(), Greater());
```

## Адаптеры контейнеров и итераторов

Адаптеры контейнеров — ограничивают интерфейс:

- stack (LIFO) — по умолчанию использует deque.
- queue (FIFO).
- priority_queue (куча).

Итераторные адаптеры:

- back_inserter, front_inserter, inserter — позволяют алгоритмам добавлять элементы в контейнер.
- reverse_iterator — обратный обход.

## std::vector

- Динамический массив, растущий по мере добавления элементов.
- Автоматическое управление памятью (конструкторы, деструктор, копирование).
- Богатый набор методов: доступ, добавление, удаление, работа с размером и ёмкостью.
- Совместимость с алгоритмами STL через итераторы.

Когда нужно хранить последовательность элементов, часто обращаться по индексу, добавлять/удалять с конца, редко — в середину. \
vector — это обёртка над динамическим массивом, которая следит за его размером и ёмкостью (capacity).

Вопросы устройства

std::vector<T> — шаблонный класс, хранящий элементы типа T в непрерывной области памяти (как обычный массив). \
Размер (size) — количество фактически хранящихся элементов. \
Ёмкость (capacity) - сколько элементов можно разместить без перевыделения памяти. \
Элементы в vector хранятся в памяти последовательно, как в массиве. Поэтому доступ по индексу O(1), а итераторы — итераторы произвольного доступа. \

Когда ёмкость превышена (например, push_back при size == capacity), vector:
- Выделяет новую область памяти бóльшего размера (обычно в 1.5–2 раза больше текущей ёмкости, зависит от реализации).
- Перемещает (или копирует) существующие элементы в новую память.
- Уничтожает старые элементы и освобождает старую память.
- Это гарантирует амортизированное O(1) для push_back — т.е. в среднем время добавления элемента не зависит от размера вектора.

Сложность основных операций

|Операция	|Сложность(время)	|Примечание|
|-----------|----------         |----------|
|Доступ по индексу ([], at)|	O(1)|	Константа
|Добавление в конец (push_back)|	Амортизированное O(1)|	Иногда O(n) при расширении|
|Удаление с конца (pop_back)	|O(1)	|Без переаллокации|
|Вставка/удаление в середине	|O(n)	|Сдвиг элементов|
|Поиск значения (линейный)	|O(n)	|Нет встроенного бинарного поиска|
|Вставка в начало	|O(n)	|Неэффективно, используйте deque|
|Изменение размера (resize)	|O(n)|	Создание/удаление элементов|
|Резервирование памяти (reserve)|	O(n)| при росте? Нет, O(0) для reserve, если new_cap > capacity, то O(n) на перемещение|

### Основные методы и примеры использования

#### Создание и инициализация

```
#include <vector>
#include <string>
using namespace std;

// Пустой вектор
vector<int> v1;

// Вектор из 5 элементов, инициализированных нулём
vector<int> v2(5);          // {0,0,0,0,0}

// Вектор из 5 элементов, инициализированных значением 42
vector<int> v3(5, 42);      // {42,42,42,42,42}

// Инициализация списком 
vector<int> v4 = {1, 2, 3, 4};

// Копирование
vector<int> v5(v4);         // копия

// Перемещение 
vector<int> v6 = std::move(v4);  // v4 теперь пуст

// Из диапазона итераторов
int arr[] = {10,20,30};
vector<int> v7(arr, arr+3);
```

#### Доступ к элементам

```
vector<int> v = {10, 20, 30, 40};

// operator[] — без проверки границ
int a = v[2];      // 30

// at() — с проверкой (бросает std::out_of_range)
int b = v.at(2);   // 30
// v.at(10) -> исключение

// front() / back()
int first = v.front(); // 10
int last = v.back();   // 40

// Получение указателя на внутренний массив (C++11)
int* ptr = v.data();   // можно использовать как массив
```

#### Размер и ёмкость

```
vector<int> v;
cout << v.size() << " " << v.capacity(); // 0 0

v.push_back(1);
cout << v.size(); // 1
cout << v.capacity(); // зависит от реализации, обычно 1 или больше

v.reserve(100);   // зарезервировать память на 100 элементов
cout << v.capacity(); // >= 100
cout << v.size(); // 1 (размер не изменился)

v.shrink_to_fit(); //запрос на уменьшение capacity до size (необязателен)
cout << v.capacity(); // может стать 1

// resize — меняет количество элементов
v.resize(5);      // добавляет 4 нуля (или default-инициализированных)
v.resize(10, 42); // добавляет 6 элементов со значением 42
v.resize(2);      // удаляет лишние элементы с конца
```

#### Добавление и удаление элементов

```
vector<int> v = {1, 2, 3};

// Добавление в конец
v.push_back(4);          // {1,2,3,4}

// Добавление в конец с конструированием на месте (C++11)
v.emplace_back(5);       // {1,2,3,4,5} (эффективнее push_back для сложных типов)

// Вставка в середину
v.insert(v.begin() + 2, 99);   // {1,2,99,3,4,5}
v.insert(v.begin(), 100);      // в начало: {100,1,2,99,...}

// Вставка нескольких копий
v.insert(v.end(), 3, 88);      // три 88 в конец

// Вставка из списка или диапазона
v.insert(v.begin(), {10,20});

// Удаление с конца
v.pop_back();   // удаляет последний

// Удаление по позиции (или диапазону)
v.erase(v.begin() + 2);        // удаляет элемент по индексу 2
v.erase(v.begin(), v.begin()+3); // удаляет первые 3 элемента

// Очистка всех элементов
v.clear();      // size = 0, capacity обычно не меняется
```

#### Итераторы и циклы

```
vector<int> v = {1,2,3,4};

// Через индексы
for (size_t i = 0; i < v.size(); ++i) cout << v[i] << " ";

// Через итераторы (классический)
for (auto it = v.begin(); it != v.end(); ++it) cout << *it << " ";

// Диапазонный for (C++11)
for (int x : v) cout << x << " ";

// Обратный итератор
for (auto rit = v.rbegin(); rit != v.rend(); ++rit) cout << *rit << " ";
```

#### Операции сравнения и присваивания

```
vector<int> a = {1,2,3};
vector<int> b = {1,2,3};
vector<int> c = {1,2,4};

a == b; // true
a < c;  // true (лексикографическое сравнение)

// Присваивание
a = c;  // копирование
a = {5,6,7}; // присваивание списка
a.assign(5, 100); // 5 элементов 100
```

#### Особенности работы с пользовательскими типами
```
struct Person {
    string name;
    int age;
    Person(const string& n, int a) : name(n), age(a) {}
    // Если не определить конструктор копирования/перемещения, они будут сгенерированы
};

vector<Person> people;
people.emplace_back("Alice", 30); // конструирование на месте
people.push_back(Person("Bob", 25)); // временный объект + копирование/перемещение

// Если тип некопируемый (например, unique_ptr), можно использовать только перемещение
vector<unique_ptr<int>> ptrs;
ptrs.push_back(make_unique<int>(5)); // ОК, перемещение
```

При перевыделении памяти элементы должны быть перемещаемыми или копируемыми. \
Если тип noexcept перемещаемый, вектор использует перемещение для эффективности; иначе — копирование.

####  Инвалидация итераторов

Итераторы, ссылки и указатели на элементы могут стать недействительными после некоторых операций:

|Операция	|Инвалидация|
|-----------|-----------|
|reserve / resize| (при росте ёмкости)	Все итераторы, ссылки, указатели (переаллокация)|
|push_back / emplace_back| при size == capacity|	Все (переаллокация)|
|push_back / emplace_back без переаллокации	|Нет|
|insert / erase|	Итераторы после позиции вставки/удаления|
|pop_back|	Итератор на удалённый элемент (end() может стать другим)|
|clear|	Все итераторы (кроме end)|

Пример ошибки:
```
vector<int> v = {1,2,3};
auto it = v.begin();
v.push_back(4);   // может вызвать переаллокацию, it становится невалидным
cout << *it;      // НЕОПРЕДЕЛЁННОЕ ПОВЕДЕНИЕ!
```

#### Резервирование для ускорения

Если вы заранее знаете, сколько элементов будет добавлено, вызовите reserve:
```
vector<int> v;
v.reserve(1000);   // одна переаллокация
for (int i = 0; i < 1000; ++i) v.push_back(i);
```

## std::list — двусвязный список

std::list<T> — шаблонный класс, реализующий двусвязный список. Каждый узел содержит:

- значение типа T,
- указатель на следующий узел,
- указатель на предыдущий узел.

Свойства:

- Элементы не хранятся в непрерывной памяти (в отличие от vector).
- Итераторы — двунаправленные (bidirectional), поддерживают ++ и --, но не +n или [].
- Вставка и удаление не инвалидируют другие итераторы (кроме итератора на удаляемый элемент).
- Список не вызывает переаллокации всей памяти при росте — новые узлы выделяются по одному.

Сложность основных операций

|Операция	|Сложность|
|-----------|---------|
|Доступ к первому/последнему элементу|	O(1)|
|Доступ к произвольному элементу|	O(n)|
|Вставка/удаление в начале/конце|	O(1)|
|Вставка/удаление в середине (если известна позиция)|	O(1)|
|Поиск элемента по значению|	O(n)|
|Получение размера (size())|	O(1) |
|Слияние двух списков (splice)|	O(1)|

### Основные методы и примеры использования

#### Создание и инициализация

```
#include <list>
#include <string>
using namespace std;

list<int> l1;                       // пустой
list<int> l2(5, 100);               // 5 элементов со значением 100
list<int> l3 = {1,2,3,4};           // инициализация списком
list<int> l4(l3);                   // копирование
list<int> l5(move(l3));             // перемещение (l3 становится пустым)
```

#### Доступ к элементам

```
list<int> l = {10,20,30};
int first = l.front();   // 10
int last  = l.back();    // 30
// l[1] - НЕЛЬЗЯ, operator[] отсутствует
```
#### Добавление и удаление элементов

```
list<int> l = {1,2,3};

l.push_back(4);          // {1,2,3,4}
l.push_front(0);         // {0,1,2,3,4}

l.pop_back();            // {0,1,2,3}
l.pop_front();           // {1,2,3}

// Вставка по итератору
auto it = l.begin();
++it;                    // указывает на 2
l.insert(it, 99);        // {1,99,2,3}

// Вставка нескольких копий
l.insert(it, 2, 77);     // {1,99,77,77,2,3}

// Удаление по итератору
it = l.begin();
l.erase(it);             // удалили 1 -> {99,77,77,2,3}

// Удаление диапазона
l.erase(l.begin(), next(l.begin(), 2)); // удалили 99,77 -> {77,2,3}
```

#### Специфические методы list

```
list<int> l1 = {1,2,3};
list<int> l2 = {4,5,6};

// Перемещение (splice) — перекладывает узлы из одного списка в другой
auto it = l1.begin();
advance(it, 2);          // указывает на 3
l1.splice(it, l2);       // вставить все элементы l2 перед 3, l2 становится пустым
// l1: {1,2,4,5,6,3}

// Сортировка (собственный метод, эффективнее, чем общий sort для списков)
l1.sort();               // по возрастанию
l1.sort(greater<int>()); // по убыванию

// Удаление дубликатов (только для отсортированного списка)
l1.unique();             // удаляет последовательные дубликаты

// Удаление элементов по условию
l1.remove(5);            // удалить все элементы == 5
l1.remove_if([](int n) { return n % 2 == 0; }); // удалить чётные

// Слияние двух отсортированных списков
l1.merge(l2);            // l1 и l2 должны быть отсортированы, результат в l1, l2 пуст
```

#### Итераторы и циклы

```
list<int> l = {1,2,3,4,5};

// Диапазонный for
for (int x : l) cout << x << " ";

// Ручные итераторы
for (auto it = l.begin(); it != l.end(); ++it) cout << *it << " ";

// Обратный обход
for (auto rit = l.rbegin(); rit != l.rend(); ++rit) cout << *rit << " ";
```

#### Размер и ёмкость

```
list<int> l = {1,2,3,4,5};
auto it = l.begin();   // указывает на 1
auto it2 = next(it);   // указывает на 2
l.erase(it);           // it инвалидирован, но it2 остаётся валидным (указывает на 2)
cout << *it2;          // OK, выведет 2
```

#### Инвалидация итераторов

Вставка и удаление не инвалидируют другие итераторы, кроме тех, что указывают на удаляемый элемент.

```
list<int> l = {1,2,3,4,5};
auto it = l.begin();   // указывает на 1
auto it2 = next(it);   // указывает на 2
l.erase(it);           // it инвалидирован, но it2 остаётся валидным (указывает на 2)
cout << *it2;          // OK, выведет 2
```

## std::forward_list — односвязный список

Отличия от std::list

|Характеристика	|std::list (двусвязный)	|std::forward_list (односвязный)
|---------------|-----------------------|------------------------------|
|Направление обхода|	Вперёд и назад	|Только вперёд|
|Итераторы	|Bidirectional|	Forward|
|Память на узел|	Два указателя (prev/next)|	Один указатель (next)|
|push_back, pop_back, back(), size()|	Есть|	Нет|
|insert, erase|	Есть (с итератором)|	insert_after, erase_after|
|reverse|	Есть|	Есть|
|splice|	Есть несколько версий|	splice_after|

### Внутреннее устройство
Узел содержит только data и указатель на следующий узел. Головной элемент не имеет предыдущего. \
Итераторы — только forward, поддерживают ++, но не --.

#### Основные методы

```
#include <forward_list>
using namespace std;

forward_list<int> fl = {1,2,3,4};

// Доступ к первому элементу
int first = fl.front();   // 1
// back() отсутствует

// Добавление/удаление в начале
fl.push_front(0);         // {0,1,2,3,4}
fl.pop_front();           // {1,2,3,4}

// Вставка после итератора
auto it = fl.begin();      // указывает на 1
fl.insert_after(it, 99);   // вставить 99 после 1 -> {1,99,2,3,4}

// Вставка нескольких
fl.insert_after(it, 2, 77); // {1,77,77,99,2,3,4} (вставляет после 1, но обратите внимание: it не изменился)

// Удаление после итератора
fl.erase_after(it);        // удаляет элемент после 1 -> {1,77,99,2,3,4} (первый 77 удалён)

// Удаление диапазона после итератора
fl.erase_after(it, fl.end()); // удаляет всё после 1 -> {1}

// Специальные методы (аналоги list, но с суффиксом _after)
fl.sort();
fl.unique();
fl.remove(77);
fl.remove_if([](int n){ return n>3; });

// Слияние (оба должны быть отсортированы)
forward_list<int> fl2 = {5,6};
fl.merge(fl2);   // fl2 становится пустым

// Перемещение (splice_after)
forward_list<int> src = {10,20};
auto pos = fl.before_begin(); // итератор перед первым элементом
fl.splice_after(pos, src);    // вставить src после pos (т.е. в начало)
```
